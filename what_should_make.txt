가장 중요한 기능으로는 **RAG 에이전트가 스스로 벡터 데이터베이스에 접근해 데이터를 저장 및 삭제하고, 검색 임계값 조정을 할수 있도록 해야함.**
그외에 네이버 API를 이용한 웹서핑, 구글 API를 이용한 실시간 인기 검색어 활용이 필요함.
벡터 데이터베이스를 활용해서 사용자 질문 > 함수활용으로 바로 넘어가는것도 괜찮을것같음

>> 기존
사용자 질문 -> LLM -> Function -> LLM -> 사용자
>> 개선 방안
사용자 질문 -> Embedding -> Vector DB -> Function -> LLM -> 사용자

기대 효과
만약 Vector DB Seaching 속도가 LLM 추론 속도보다 빠르다면 속도 개선 극대화 가능

How?
여러가지 Function Calling에 대한 질문들의 유사도를 파악한다음 유사도가 높으면 함수를 실행하는 방식으로하면 될듯
